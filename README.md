# Hebrew Voice Command Reformulation API

A FastAPI backend that wraps a Hebrew NLP pipeline (AlephBERT intent classifier + heBERT NER) to
transform natural-language Hebrew utterances into structured commands suitable for reading aloud
to Siri. Designed for elderly Israeli smartphone users.

---

## Starting the Server Locally

```bash
# From the repository root (activate your Python 3.10 virtualenv first):
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

The server loads all three NLP models at startup (~10–20 seconds on first run).
Once you see `=== All models ready. Server accepting requests. ===` in the logs, the server is ready.

Interactive API docs (auto-generated by FastAPI):

```
http://localhost:8000/docs      ← Swagger UI
http://localhost:8000/redoc     ← ReDoc
```

## Testing the API from Your Browser

The browser URL bar only sends GET requests, so it cannot call `POST /reformulate` directly.
The two browser-native ways to interact with the backend are:

### 1. Health check — URL bar (GET)

Paste this URL directly into your browser address bar:

```
http://localhost:8000/health
```

You will see the JSON response immediately:

```json
{"status": "ok", "models_loaded": true}
```

### 2. Send a Hebrew command — Swagger UI (no tools needed)

FastAPI's built-in Swagger UI lets you send POST requests from the browser with no extensions.

**Step-by-step:**

1. Open `http://localhost:8000/docs` in your browser.
2. Click `POST /reformulate` to expand the section.
3. Click the **"Try it out"** button (top-right of the section).
4. Replace the example body with your Hebrew command, for example:
   ```json
   { "utterance": "תצלם תמונה" }
   ```
5. Click **"Execute"**.
6. The response appears below with the full JSON:
   ```json
   {
     "status": "success",
     "original": "תצלם תמונה",
     "intent_id": 6,
     "intent_label": "camera",
     "reformulated": "תפתחי מצלמה"
   }
   ```

You can type any Hebrew utterance into the `utterance` field and click Execute to see the
pipeline result in real time.

## API Reference

### `GET /health`

Liveness probe. Returns the server status and whether the intent classifier has been loaded.

**Response — 200 OK**

```json
{
  "status": "ok",
  "models_loaded": true
}
```

| Field | Type | Description |
|---|---|---|
| `status` | string | Always `"ok"` when the server is running. |
| `models_loaded` | boolean | `true` once the AlephBERT classifier is loaded into memory. |

**Example**

```bash
curl http://localhost:8000/health
```

```json
{
  "status": "ok",
  "models_loaded": true
}
```

---

### `POST /reformulate`

Main pipeline endpoint. Accepts a raw Hebrew utterance, validates the input character set,
classifies the intent, extracts named entities, and returns a structured Hebrew command.

**Request**

`Content-Type: application/json`

| Field | Type | Required | Description |
|---|---|---|---|
| `utterance` | string | Yes | Raw Hebrew voice command. Must contain only Hebrew letters and spaces. English letters, digits, and special characters are rejected with HTTP 400. |

**Example request body**

```json
{
  "utterance": "תשלחי הודעה לישראל שאני מאחרת"
}
```

**Response — 200 OK (pipeline executed)**

HTTP 200 is returned whenever the pipeline runs, regardless of output quality.
The `status` field distinguishes the two pipeline outcomes.

| Field | Type | Description |
|---|---|---|
| `status` | `"success"` \| `"failed"` | `"success"` when the output passed all validation checks. `"failed"` when the pipeline produced an unusable result — in that case `reformulated` is `null`. |
| `original` | string | The input utterance as received (stripped of leading/trailing whitespace). |
| `intent_id` | integer | Predicted intent class index, 0–9. |
| `intent_label` | string | Human-readable intent name (see intent table below). |
| `reformulated` | string \| null | Structured Hebrew command for Siri. `null` when `status` is `"failed"`. |

**Example — success**

```bash
curl -X POST http://localhost:8000/reformulate \
     -H "Content-Type: application/json" \
     -d '{"utterance": "תשלחי הודעה לישראל שאני מאחרת"}'
```

```json
{
  "status": "success",
  "original": "תשלחי הודעה לישראל שאני מאחרת",
  "intent_id": 2,
  "intent_label": "sms",
  "reformulated": "שלח הודעה לישראל אני מאחרת"
}
```

**Example — failed (unusable pipeline output)**

```json
{
  "status": "failed",
  "original": "תזמין שולחן לשניים",
  "intent_id": 5,
  "intent_label": "calendar",
  "reformulated": null
}
```

**Response — 400 Bad Request**

Returned when the input is empty, whitespace-only, or contains characters outside the allowed
set (Hebrew letters and spaces only).

```json
{
  "detail": "Invalid input."
}
```

**Response — 422 Unprocessable Entity**

Returned by FastAPI when the request body is malformed or the `utterance` field is missing.

```json
{
  "detail": [
    {
      "type": "missing",
      "loc": ["body", "utterance"],
      "msg": "Field required"
    }
  ]
}
```

**Response — 500 Internal Server Error**

Returned when the reformulation module raises an unexpected exception. The error message is
intentionally generic — internal details are logged server-side only.

```json
{
  "detail": "An internal error occurred."
}
```

| 1 | `alarm` | Set an alarm |
| 2 | `sms` | Send a text message |
| 3 | `search_query` | Perform a web search |
| 4 | `navigation` | Get directions |
| 5 | `calendar` | Create a calendar event |

| 7 | `weather` | Check the weather |
| 8 | `notes` | Create a note |
| 9 | `flashlight` | Toggle the flashlight |

## Input Validation Rules

Both the input utterance and the pipeline output are validated against the same character set,
derived from the command templates:

- **Allowed:** Hebrew letters (Unicode U+05D0–U+05EA) and ASCII spaces.
- **Rejected:** English letters, digits, punctuation (`,`, `:`, `-`, `?`), and any other character.

Validation runs in two stages:
1. **Stage 1 (input):** Before the pipeline. Invalid input → HTTP 400.
2. **Stage 2 (output):** After reformulation. Unusable output → `status: "failed"`, HTTP 200.
